• ¿Qué es GitHub?

    GitHub es un servicio de alojamiento de repositorios en donde los desarrolladores pueden gguardar y gestionar sus distintos proyectos y software.
Es una herramienta gratuita y de codigo abierto, lo que permite poder trabajar con otros usuarios en un mismo proyectos

• ¿Cuáles son los beneficios de usar GitHub?

    las facilidades que nos da github son varias:
    
    -Es una herramienta facil de usar y de aprender, con lo que es muy faciel familiarizarce para hacer los controles de versiones

    -GitHub puede ir escalando dependiendo el tipo de proyecto, siendo uno personal como uno mas grande de empresas 

    -Ofrece una variedad de caracteristicas para garantizar la seguridad de los proyectos

    - Cuenta con una gran comunidad de desarrolladores 

• ¿Cuáles son los tipos de ramas en Git?  

en git existen diversos tipos de ramas, algunas son:

    -Rama Principal (main o master)
    -Ramas de Desarrollo
    -Ramas de Características
    -Ramas de Corrección de Errores
    -Ramas de Soporte
    -Ramas de Lanzamiento 
    -Ramas de Experimento

• ¿Cómo crear una rama en Git?
    
    Para crear una rama en git premero abrimos la termilanal y ahi utilizamos el coamando git branch 

• ¿Cómo cambiar a una rama en Git?

    para cambiar una rama usamos el coamando git checkout desde la termilanal

• ¿Cómo fusionar ramas en Git?

    para fusionar una rama usamos el coamando git merge desde la termilanal

• ¿Cómo crear un commit en Git?

    para crear un commit usamos el coamando git commit -m ""  desde la termilanal 

• ¿Cómo enviar un commit a GitHub?

    para enviar un commit usamos el coamando "push" en la termilanal

• ¿Qué es un repositorio remoto?

    Es un directorio localizado en tu computadora donde almacenas tu codigo al que le instalas git 

• ¿Cómo agregar un repositorio remoto a Git?

    Primero es necesario obtener la url del repositorio,  despues agregar el repositorio remoto 
con git remote add nombre-remoto URL-del-repositorio y verificar el repositorio remoto con git remote -v.
    
• ¿Cómo empujar cambios a un repositorio remoto?

    Para empujar los cambios utilizamos  el comando git push nombre_del_proyecto en la termilanal

• ¿Cómo tirar de cambios de un repositorio remoto?

    Primero asegúrarnos de estar en la rama correcta con git checkout nombre-de-la-rama o nombre-del-archivo.
Despues tira los cambios del repositorio remoto con el comando git pull origin nombre-de-la-rama y verifica 
el estado del repositorio con git status.

• ¿Qué es un fork de repositorio?

    Fork es lo que llamamos hacer una copia a repositorio localizado en GitHub o en otra plataforma de almacenamiento de codigo,
    para hacerlo dando click en Fork situado en la parte superior derecha de github

• ¿Cómo crear un fork de un repositorio?

    Para crear un fork  para hacerlo  hay que darle click en a la pestaña Fork situado en la parte superior derecha de github

• ¿Cómo enviar una solicitud de extracción (pull request) a un repositorio?

    Para enviar una solicitud  nos dirigiremos a la solapa de Pull requests allí daremos click en new
pull request, le damos click en Create pull request, ahi podemos escribir un texto explicando los cambios
que hicimos a la compia en nuestro dispositivo 

• ¿Cómo aceptar una solicitud de extracción?

   Para aceptar una pull request nos dirigiremos a la pestaña con el mismo nombre y ahi podemos ver todas las solisitudes que tenemos y podemos
elegimos la solicitud que deseamos aceptar
    
• ¿Qué es un etiqueta en Git?

    Una etiqueta en git es una forma de marcar puntos específicos del historial de commits como puntos
importantes, es otra forma de hacerle seguimiento a nuestro proyecto

• ¿Cómo crear una etiqueta en Git?

    Para crear una etiqueta usamos el coamando git tag en la termilanal 

• ¿Cómo enviar una etiqueta a GitHub?

    Para enviar tenemos que usar el coamando git push nombre-etiqueta

• ¿Qué es un historial de Git?

    El historial en Git es una característica fundamental que nos sirve para rastrear todos los cambios 
realizados en un repositorio a lo largo del tiempo

• ¿Cómo ver el historial de Git?

    Para vere el historial usamos el coamando git log

• ¿Cómo buscar en el historial de Git?

    Para buscar en el historial usamos el comando git log -- path/al/archivo

• ¿Cómo borrar el historial de Git?

    Para borrar el historial usamos el coamando  git stash --all
